基础 · 万能提示
一、万能通用提示词
CORE · 01

建议每次对话开头都先用一个“任务说明模版”，告诉模型：你是谁、要做什么、输出格式、风格和深度。

1
0.1 通用任务说明模版
›
模版：
你现在是一个【角色，例如：资深财务顾问 / 教学型助手 / 严谨的编辑】。

请帮我完成这个任务：
【用 2–3 句话说明你想要什么：例如“帮我理解 XXX，并做一份学习计划”】

要求：
1. 输出结构：请用【列表 / Markdown 表格 / 分步骤】输出。
2. 目标对象：假设我是【完全新手 / 有一定基础 / 专业人士】，解释难度要匹配。
3. 风格：整体语气【正式 / 口语 / 简洁 / 详细】。
4. 深度：先给一个整体框架，再一步步展开，每一步都要有例子或类比。
5. 如有不清晰的地方，你可以根据常识自行合理假设，不用反问我。

用法：每次开始一个新任务时先发这个模版，再接问题。

学习 · 概念理解
二、学习 / 理解复杂概念
LEARN · 02

适合用来啃难概念、让模型当老师、出题纠错，形成“教 + 练 + 反馈”的闭环。

2
1.1 拆解一个概念 + 举例 + 对比
›
模版：
请用“教新手”的方式，帮我理解【概念，例如：通货膨胀 / 贴现率 / PLC 扫描周期】。

要求：
1. 先用 1–2 句话给出“最精简”的定义。
2. 用 3–5 点，从“它是什么 / 为什么重要 / 在哪用得上”来解释。
3. 用一个贴近日常生活的类比说明这个概念。
4. 选 2–3 个容易混淆的概念（如果有），做一个对比表：列出“相同点 / 不同点 / 适用场景”。
5. 最后给我 3 个自测问题，用来检查我是不是真的理解了。
3
1.2 让模型当你的私人老师（分阶段教学）
›
模版：
你现在是我的【学科，例如：宏观经济 / 股票投资 / 电气工程基础】私人老师。

我的基础是：【零基础 / 只懂一点皮毛 / 有工作经验】。

请：
1. 先给我一份“从零到入门”的知识地图（用树状或分级列表）。
2. 按照 3 个阶段拆分学习：（入门 → 进阶 → 巩固），每个阶段给：
   - 学习目标
   - 关键概念列表
   - 推荐练习方式
3. 给出一套【7 天】的学习计划：每天学什么、练什么，大概需要多久。
4. 每天末尾设计 3–5 个小题目，让我自测。
4
1.3 问答式辅导 + 纠错
›
模版：
接下来请你用“出题 + 纠错”的方式教我【主题】。

规则：
1. 你先问我一个难度适中、与【主题】相关的问题。
2. 等我回答后，请你：
   - 判断是否正确
   - 指出具体错误点
   - 给出标准答案和简短解释
3. 根据我的表现，逐步提高或降低难度。
4. 整个过程像自适应练习一样进行。
（现在请先出第 1 题，并说明考察的知识点。）
总结 · 做笔记
三、阅读 / 总结 / 做笔记
NOTE · 03

用来处理长文章、报告、论文、聊天记录，帮你变成结构化笔记和逻辑图。

5
2.1 长文总结 + 多层级笔记
›
模版：
下面是我要阅读的内容（可能比较长）： 
【粘贴文章 / 报告 / 对话】

请帮我做“学习型笔记”，要求：
1. 先用 3–5 句话写一个“给外行看的摘要”。
2. 再用分层结构（一级标题、二级小点）整理出知识框架。
3. 用表格列出：每个关键概念 → 一句话解释 → 文中对应的重要结论。
4. 最后给我 5 个讨论问题，帮助我在之后继续思考和延伸。
6
2.2 把逻辑结构画出来
›
模版：
请帮我分析下面这段内容的逻辑结构：
【粘贴内容】

输出：
1. 用“因果链”的形式写出：原因 → 过程 → 结果 → 影响。
2. 如果逻辑里有前提假设，请单独列出来，并指出是否合理。
3. 用简单的项目符号画出“逻辑流程图”（用文字模拟流程图即可）。
写作 · 改写 · 翻译
四、写作 / 改写 / 翻译
WRITE · 04

先自己写，再让模型帮你改结构、改表达、改逻辑，是最稳妥的用法。

7
3.1 提升一段文字的质量（保持原意）
›
模版：
下面是一段我自己写的内容，用于【场景，例如：求职简历 / 工作汇报 / 电邮】：

【粘贴原文】

请帮我：
1. 在不改变核心信息的前提下，重写一版，更【正式 / 简洁 / 有说服力】。
2. 结构上更清晰，适当分段，并加上小标题（如果合适）。
3. 指出你做了哪些主要修改（用要点列出：例如“语气更专业”“逻辑顺序调整”等）。
4. 如果有明显信息缺失，请列出建议我补充的点，而不是自己瞎编。
8
3.2 先定结构再扩写
›
模版：
我想写一篇关于【主题，例如：我这一年的成长 / 某个项目总结 / 某个知识点】的文章，目标读者是【对象】，大概【字数】。

请：
1. 根据目的和读者，先帮我设计一个结构清晰的大纲（3–5 个一级标题，每个 2–3 个要点）。
2. 每个要点下面，给出“我可以写的具体内容提示”（例如经验、案例、数据等）。
3. 在我确认大纲后，再按大纲逐段帮我扩写。
（先不要直接写全文，请先给大纲。）
9
3.3 多种语气版本
›
模版：
下面这段话请帮我改写成 3 个版本：正式商务 / 朋友聊天式 / 简洁干练式。

原文：
【粘贴内容】

要求：
1. 不要改变事实信息。
2. 仅调整语气、措辞和长短句的节奏。
10
3.4 保留语气的翻译（中 ↔ 英）
›
模版：
请把下面这段【中文/英文】翻译成【目标语言】，场景是【求职邮件 / 工作汇报 / 学术写作 / 轻松聊天】。

原文：
【粘贴内容】

要求：
1. 保留原文的语气和礼貌程度。
2. 适当本地化表达，不要逐字直译。
3. 如果某些地方原文表达不够自然，可以顺手润色一下，并列出你做的主要调整。
工作 · 文档场景
五、工作 / 文档 / 报告
WORK · 05

项目复盘、工作总结、会议纪要、简历优化，这一组可以直接变成日常工作流。

11
4.1 项目复盘 / 工作总结
›
模版：
我想写一篇【项目复盘 / 月度总结 / 年度总结】，对象是【领导 / 同事 / HR】，内容大致如下：
【简单描述项目或这一段时间做的事】

请帮我：
1. 按“背景（B）–任务（T）–行动（A）–结果（R）–反思（L）”五部分设计结构。
2. 在每一部分下帮我列出可以写的要点提示（用问题形式提醒我补充信息）。
3. 在我补充完信息后，再帮我生成可直接使用的一篇完整文字。
12
4.2 会议纪要 / 讨论整理
›
模版：
下面是我们开会时的零散记录：
【粘贴会议记录或聊天记录】

请整理成一份结构化的会议纪要：
1. 会议主题、时间、参与人。
2. 讨论问题列表（每个问题下用要点整理主要观点）。
3. 已形成的决策 / 结论。
4. 待办事项（每条包含：负责人、deadline、产出物）。
13
4.3 简历优化提示词
›
模版：
下面是我简历中的一段经历：
【粘贴经历】

请按“STAR 法 + 可量化结果”帮我优化：
1. 先分析这段经历目前的问题（例如：太笼统 / 不具体 / 没有结果）。
2. 用 STAR（情境、任务、行动、结果）的结构帮我重写这段经历。
3. 尽量用数字、规模、效率提升等方式把结果量化。
4. 示例语言保持【中文 / 英文】，风格偏【正式 / 简洁】。
技术 · 编程辅助
六、编程 / 技术类
CODE · 06

即使没有编程基础，也可以用自然语言描述需求，让模型写示例代码、解释报错。

14
5.1 用自然语言描述需求 → 生成示例代码
›
模版：
我在学【语言/技术栈，例如 Python / C / PLC 编程】，现在想实现这个功能：
【用自然语言描述你想做什么】

请：
1. 先确认并补全需求，如果你觉得有不明确的地方，可以自行做合理假设并说明。
2. 给出一段清晰的示例代码，并加上必要的注释。
3. 用通俗语言解释这段代码的整体思路和关键语句。
4. 提出 2–3 个小练习，帮助我在这个基础上改进或扩展功能。
15
5.2 让模型帮你读代码
›
模版：
下面是一段我看不太懂的代码（语言：【语言】）：

【粘贴代码】

请帮我：
1. 概括这段代码整体是做什么的，用 3–5 句话说明。
2. 分块解释：按逻辑模块或函数拆开，每块说明目的和流程。
3. 指出潜在的问题或风险（如果有）。
4. 提一个“如何改写得更清晰”的建议，并给一个改写示例。
16
5.3 bug 分析提示词
›
模版：
我在运行下面这段代码时遇到错误：

代码：
【粘贴代码】

错误信息：
【粘贴报错】

请：
1. 帮我定位大概是哪一部分逻辑导致的错误。
2. 用新手能理解的方式解释错误的根本原因。
3. 给出修改后的版本，并标出改动点。
4. 告诉我今后如何避免类似错误。
决策 · 深度思考
七、思考 / 决策 / 方案对比
DECIDE · 07

把模型当“理性顾问”：帮你拆决策维度、做方案对比、找薄弱假设。

17
6.1 系统决策分析模版
›
模版：
我现在需要在几个选项之间做选择，主题是：【例如 换工作 / 学什么专业 / 选择哪种投资方式】。

我的备选方案有：
A. 【方案 A】
B. 【方案 B】
C. 【方案 C】（如果有）

请：
1. 帮我列出“做这个决定”时最关键的 3–5 个考量维度（例如收入、发展、风险、兴趣等）。
2. 用表格的方式，按这些维度对各个方案做对比（可以用“高/中/低 + 简短说明”的形式评价）。
3. 提出你认为我容易忽略的隐性成本或风险。
4. 不需要直接替我做决定，但可以给一套“如何做出适合自己的选择”的步骤建议。
18
6.2 质疑我的想法（反向检验）
›
模版：
我现在有一个想法 / 结论：
【写下你的观点或计划】

请你扮演“温和但严格的反对者”，帮我：
1. 找出这个想法中最薄弱的 3–5 个假设或前提。
2. 针对每个薄弱点，给出可能出问题的情景和后果。
3. 提出一些我可以收集的证据或数据，用来验证这些假设。
4. 在以上基础上，帮我稍微优化这个想法，使它更稳健一些。
规划 · 习惯 · 学习
八、规划 / 学习计划 / 习惯
PLAN · 08

学什么、怎么学、多长时间学完，都可以交给模型先帮你“倒推拆解”。

19
7.1 制定学习计划
›
模版：
我想在【时间，例如 21 天 / 3 个月】内，在每天【1–2 小时】的时间里，系统学习【主题，例如：宏观经济 / 股票入门 / PLC / 英语口语】。

我的基础是：【简单说明自己的水平】。

请：
1. 先给出这段时间能达到的“现实但有挑战”的目标。
2. 把学习分成几个阶段（例如：基础概念 → 典型案例 → 实战练习）。
3. 制定一个按天/按周的学习计划表：每天/每周的任务、建议时长、对应资源类型（书/视频/练习）。
4. 给出一个“复盘模版”，告诉我每周结束时应该怎么检查自己的进度和理解。
20
7.2 倒推目标
›
模版：
我的目标是：【写清楚，例如 1 年内找到测试工程师职位 / 半年内能看懂财报 / 3 个月内体重增加 5kg】。

请帮我：
1. 从目标倒推，拆成 3–5 个关键里程碑（时间节点 + 能力/成果）。
2. 每个里程碑下，列出必须完成的行动（习得的技能 / 必须做的项目 / 可验证指标）。
3. 提出可能阻碍我完成这些里程碑的风险，并给几个应对策略。
创意 · 头脑风暴
九、创意 / 头脑风暴
IDEA · 09

命名、策划活动、想功能、做店铺品牌，可以先大规模发散，再收敛。

21
8.1 批量生成创意 + 分类
›
模版：
我需要围绕【主题，例如：店铺名字 / 课程名字 / 活动方案 / 产品功能】进行头脑风暴。

请：
1. 先帮我明确约束条件（例如目标人群、风格、长度限制等）。
2. 在此基础上，给我至少【20】个创意，并按【风格/定位】分类展示（用表格）。
3. 每个创意后附一句话说明它的定位或亮点。
4. 最后根据这些创意，总结出 3–5 条命名/设计的“潜在规律”。
反思 · 元认知
十、个人反思 / 元认知（温和版）
REFLECT · 10

帮你把“脑子里的糊糊”变成结构化的事实-想法-感受，再找新的视角和行动。

22
9.1 梳理一件事：事实-想法-感受
›
模版：
我想梳理一件让我有点困扰的事情：

【简要描述发生了什么】

请帮我：
1. 用“事实 – 我的想法 – 我的情绪”三栏，帮助我把这件事拆开。
2. 帮我识别我的想法里是否有典型的“思维偏差”（如果有，请简单说明是什么、为什么）。
3. 提出 3–5 个更平衡、更加全面的替代性看法（不是说服我，而是给我更多视角）。
4. 给一个简单的行动建议：接下来 1–2 步，我可以做什么。
元设定 · 提升回答质量
十一、元提示词：让回答更好用
META · 11

通用“回答规范模版”，可以贴在任何问题前面，让输出更结构化、可复用。

23
10.1 要求“先给框架，再展开”和“自查”
›
模版：
请按以下流程回答我的任何问题：

1. 先给一个不超过 5 行的“结论概览”（用列表）。
2. 再给出一个结构化的大纲（1、1.1、1.2 的形式）。
3. 按大纲逐点展开，每一点都尽量有例子或具体做法。
4. 最后用 3–5 个要点自查你的回答：是否有遗漏常见角度？是否有可以更具体的地方？请在最后一小节写出“改进建议”，告诉我如果我愿意，我们可以在哪些点继续深入。

接下来，我的问题是：
【写上你的问题】
